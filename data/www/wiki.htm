URL's that are supported by the ESP Chime

      Chime ring
- When user/password empty: http://%ipaddr%/ring
      Chime custom ring
This option can be used to send a http string with the data, required data:
   • rcswitch library supported protocol
   • rcswitch library supported pulse
   • rcswitch library supported code
- When user/password empty: http://%ipaddr%/customring?protocol=2&pulse=712&code=00110000101111001101010110010011
      Chime reboot
- When user/password empty: http://%ipaddr%/reboot

  Page: ESP Info

    ESP Info

      Versionbuild:
- The ESP Chime firmware and date.
      Wifi SSID:
- To which SSID is the ESP Chime connected and what is it's RSSI Value.
      Ip:
- The IP address, Subnet mask, Default gateway.
      CAM:
- Which board is selected, the GPIO-pinnumber for the RCSwitch, the GPIO-pinnumber for de photomos.
      Spiffs:
- The maximum, used and free memory for spiffs.
      Server:
- Sending protocol, IP address server en port number of the server.

    ESP Logging

      DumpLog
- Saves the log to a file, on the device which is logged-in.
      CleanLog
- Cleans the log-field.

  Page: ESP Settings

      WebLog Display Level:
- Sets the level of log.
(None, Error, Warning, Info, Debug)
      ESP name:
- Sets the name of the device.
      Web Userid:
- Login name for secure login on ESP-chime. If security is not needed, clear this field.
      Password:
- Login password for secure login on ESP-chime. If security is not needed, clear this field.
      Network:
☛ Selected "DHCP" to get network credentials from network.
☛ Select "Fixed" to set your own netwerk credentials.
          IP Address:
    - Set the IP for the ESP chime.
          Subnetmask:
    - Set the subnetmask for the ESP chime.
          Gateway Addr:
    - Set the gateway address for the ESP chime.
          DNS IPAddress:
    - Set the DNS address for the ESP chime.
      RF transmit settings:
☛ Only settings from the RCswitch library can be used (https://github.com/sui77/rc-switch)
          RF Protocol:
    - The number for the RF protocol, see RCswitch library for the number.
          RF Pulse:
    - The pulse length for the code.
          RF Code:
    - The code itself (1 or 0), max 32 bit long.
      Send Protocol:
☛ Sets the protocol how to send chime URL received to server
(none, json, mqtt)
          Server IP:
    - Set the IP for the server, where the message needs to be send to.
     (This function is the same for json and mqtt)
          Server Port:
    - Set the port number for the server, where the message needs to be send to.
     (This function is the same for json and mqtt)
          Server Userid:
    - When using secure connection, user ID/name can be added.
     (This optional, is the same for json and mqtt)
          Server Password:
    - When using secure connection, password can be added.
     (This optional, is the same for json and mqtt)
          Domoticz IDX:
    - The Domoticz IDX number of the device which needs to be controlled by ESP Chime.
     (This function is the same for json and mqtt)
          Send IDX OFF:
    - Is de ESP Chime needs to send an OFF command to Domoticz.
     (This function is the same for json and mqtt)
          MQTT subscriber:
    - The MQTT subscriber name to receive the mqtt messages.
          MQTT TopicIn::
    - The MQTT topic name, when sending mqtt messages.
